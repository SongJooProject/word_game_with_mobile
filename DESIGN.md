# DESIGN.MD - UI/UX 디자인 가이드라인

## 개요
법률 용어 학습 게임의 UI/UX 디자인 규칙 및 레이아웃 정의

---

## 레이아웃 구조

### 메인 레이아웃 (한 페이지 메뉴)
```
+------------------+--------------------------------------------+
|                  |  과목 이름                    | 점수   |
|   사이드바       |--------------------------------------------|
|   (과목 선택)    |                                            |
|                  |  +--------------------------------------+ |
|                  |  | 챕터 1                               | |
|                  |  | [전체] [섹션1] [섹션2] [섹션3]       | |
|                  |  +--------------------------------------+ |
|                  |  +--------------------------------------+ |
|                  |  | 챕터 2                               | |
|                  |  | [전체] [섹션1] [섹션2]               | |
|                  |  +--------------------------------------+ |
|                  |                                            |
+------------------+--------------------------------------------+
```

### 화면 전환 순서
1. **과목 선택** → 사이드바에서 과목 클릭
2. **챕터 선택** → "과목을 선택하세요" → 챕터 목록
3. **섹션 선택** → 챕터 이름 → 섹션 목록 (전체 포함)
4. **게임 유형 선택** → 선택형/빈칸형/전체
5. **게임 진행** → 문제 표시 → 정답 확인 → 다음 문제
6. **결과** → 최종 점수 표시

---

## 메뉴 시스템

### 사이드바
- **위치**: 왼쪽 고정 (180px)
- **배경**: 반투명 흰색 (`rgba(255, 255, 255, 0.95)`)
- **과목 목록**: 세로 스택 배치

### 메인 메뉴 영역 (한 페이지)
- **레이아웃**: 모든 챕터와 섹션이 한 페이지에 표시
- **챕터 블록**: 각 챕터마다 카드 형태로 표시
- **섹션 버튼**: 가로 배치 (flex-wrap)
- **전체 선택**: 초록색 버튼으로 강조

### 메뉴 구조
```
형사소송법
├─ 챕터 1: 관할 및 법원의 결정, 판결
│  ├─ [전체] (28문제)
│  ├─ [사물관할]
│  ├─ [관할위반 및 이중기소 처리]
│  └─ [토지관할]
└─ 챕터 2: 제척, 기피
   ├─ [전체] (26문제)
   ├─ [제척사유]
   └─ [기피사유]
```

## 문제 표시

### 선택형 (Type 1)
```
+--------------------------------------------------+
| 1번                                    [선택형]   |
+--------------------------------------------------+
|                                                  |
|  문제 텍스트가 여기에 표시됩니다                   |
|                                                  |
+--------------------------------------------------+
|  1. 선택지 1                                      |
|  2. 선택지 2                                      |
|  3. 선택지 3                                      |
|  4. 선택지 4                                      |
+--------------------------------------------------+
|          [정답 확인]                              |
+--------------------------------------------------+
```

### 빈칸형 (Type 2)
```
+--------------------------------------------------+
| 1번                                    [빈칸형]   |
+--------------------------------------------------+
|                                                  |
|  문제 텍스트가 여기에 (1) 표시됩니다              |
|                                                  |
+--------------------------------------------------+
|  [정답 입력]                                     |
+--------------------------------------------------+
|          [정답 확인]                              |
+--------------------------------------------------+
```

## 타이포그래피

### 폰트
- **기본**: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Noto Sans KR', sans-serif
- **제목**: 1.75em, 볼드
- **챕터 헤더**: 1.1em, 볼드
- **본문**: 1.1em
- **작은 텍스트**: 0.9em

### 간격
- **메뉴 아이템 간격**: 1px (최소)
- **챕터 블록 간격**: 24px
- **섹션 버튼 간격**: 8px
- **메인 패딩**: 24px 40px

## 반응형 디자인

### 데스크톱 (768px 초과)
- 사이드바 고정 표시
- 메뉴 영역 중앙 정렬

### 모바일 (768px 이하)
- 사이드바 토글 버튼으로 제어
- 메뉴 전체 너비 사용
- 버튼 세로 스택

---

## 애니메이션

### 전환 효과
- **슬라이드 인**: `0.3s ease` (문제 카드)
- **페이드 인**: `0.2s ease` (메시지)

### 인터랙션
- **호버**: 배경색 변화 (0.15s ease)
- **포커스**: 블루 테두리 + 그림자

---

## 접근성

### 키보드 단축키
- **Enter**: 다음 동작 (정답 확인 또는 다음 문제)
- **Escape**: 뒤로 가기 또는 사이드바 토그
- **Backspace**: 뒤로 가기

### 시각적 피드백
- **정답**: 초록색 테두리 + 배경
- **오답**: 빨간색 테두리 + 배경 + 쉐이크
- **호버**: 색상 변화 + 약간의 이동
