# DESIGN.MD - word-game 디자인 가이드라인

## 개요
법률 용어 빈칸 채우기 게임의 디자인 사양을 관리합니다.

## 할루시네이션 방지 규칙
- 문제 데이터는 `data/questions.json` 파일에서만 관리
- 하드코딩된 문제 금지
- 법률 전문가 검수 후 문제 등록
- AI가 생성한 문제 사용 시 반드시 검증 필요
- 법령용어 수집은 `LAWDB.MD` 참조

---

## UX 사양

### 1. 게임 플로우
```
주제 선택 → 문제 유형 선택 → 게임 시작 → 문제 풀기 → 정답 피드백 → 다음 문제 → 결과 화면
```

#### 단계별 상세
1. **주제 선택**: 사이드바에서 주제 클릭 → 하이라이트
2. **문제 유형 선택**: OX 문제 또는 빈칸 채우기 선택
3. **게임 시작**: 로딩 애니메이션 → 첫 문제 표시
3. **문제 풀기**: 입력 포커스 → Enter로 제출
4. **정답 피드백**: 
   - 정답: 초록색 하이라이트 + "정답!" 메시지 + 점수 증가 애니메이션
   - 오답: 빨간색 테두리 + "틀렸습니다" + 남은 기회 표시
5. **다음 문제**: 슬라이드 전환 애니메이션
6. **결과 화면**: 최종 점수 + 통계 + 다시 하기 버튼

### 2. 피드백 시스템

#### 시각적 피드백
| 상태 | 색상 | 애니메이션 |
|------|------|-----------|
| 입력 중 | 블루 테두리 | 펄스 효과 |
| 정답 | 민트 (#4ecca3) | 팝업 효과 |
| 오답 | 레드 (#e94560) | 흔들림 효과 |
| 5회 오답 | 오토 표시 | 페이드 인 |

#### 점수 시스템
- 정답: +10점
- 연속 정답: +5점 보너스
- 5회 오답 시 자동 다음 문제 (감점 없음)

### 3. 진행 상황 표시

#### 프로그레스바
- 위치: 상단 헤더下方
- 색상: 그라데이션 (#533483 → #e94560)
- 애니메이션: 부드러운 채우기

#### 통계
- 총 문제 수
- 풀은 문제 수
- 정답률
- 소요 시간

### 4. 애니메이션

#### 전환 효과
- 문제 전환: 슬라이드 좌→우 (0.3s)
- 메시지 표시: 페이드 인 (0.2s)
- 점수 변경: 숫자 스크롤 (0.5s)

#### 인터랙션
- 버튼 호버: translateY(-3px) + 그림자
- 입력 포커스: 테두리 빛남
- 주제 선택: 좌측 슬라이드

### 5. 반응형

#### 브레이크포인트
- 데스크톱: 1024px 이상 (사이드바 200px)
- 태블릿: 768px~1024px (사이드바 150px)
- 모바일: 768px 미만 (사이드바 접힘)

#### 모바일 대응
- 사이드바: 상단 드롭다운으로 변경
- 입력 필드: 풀 너비
- 버튼: 풀 너비

### 6. 접근성

#### 키보드 네비게이션
- Tab: 입력 필드 간 이동
- Enter: 제출/다음
- Esc: 메뉴 열기

#### 스크린 리더
- aria-label 속성
- 역할(role) 정의
- 알림 영역(live region)
